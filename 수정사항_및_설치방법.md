# PRC Running App 수정사항 및 설치방법

## 반영된 수정사항

1. 달력 기본 월 고정 문제 수정
   - 기존: 2026년 5월로 고정
   - 수정: 접속한 현재 날짜 기준 년/월 자동 표시
   - 대상: 메인 일정 달력, Dashboard 월 선택

2. 일정 등록 기본 날짜 수정
   - 기존: 2026-05-20 고정
   - 수정: 오늘 날짜 자동 입력

3. Feed 업로드 오류 메시지 개선
   - 이미지 10MB, 동영상 80MB 초과 여부를 업로드 전에 먼저 검사
   - Supabase 환경변수 누락, bucket 없음, policy 권한 문제를 구분해서 안내
   - 2.8MB 이미지처럼 용량 문제가 아닌데도 용량 오류처럼 보이는 메시지 개선

4. Gallery 업로드 오류 메시지 개선
   - 이미지 10MB 초과 여부 사전 검사
   - Supabase bucket/policy 문제를 더 명확히 표시

5. 빌드 검증 완료
   - `npm run build` 성공 확인

## 압축 파일 구성 안내

이 압축 파일에는 `node_modules`와 `.git` 폴더를 제외했습니다.
이 두 폴더를 포함하면 Windows/Vercel/Linux 환경에서 실행 권한 또는 Rollup optional dependency 오류가 발생할 수 있습니다.

## 설치 방법

```bash
npm install
npm run build
npm run dev
```

## Vercel 환경변수 필수

Vercel Project Settings > Environment Variables에 아래 값을 등록해야 Supabase Storage 업로드가 됩니다.

```env
VITE_SUPABASE_URL=본인 Supabase Project URL
VITE_SUPABASE_ANON_KEY=본인 Supabase anon public key
```

## Supabase Storage bucket 필요

아래 bucket이 필요합니다.

- feed
- gallery
- profile
- hero

`docs/supabase-storage-setup.sql` 파일의 SQL을 Supabase SQL Editor에서 실행해 Storage Policy를 설정하세요.
